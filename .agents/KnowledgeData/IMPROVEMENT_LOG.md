# 波段交易系統改進報告

## 目前問題

### 1. 數據不足
- CoinGecko 免費版只提供 23-31 天 OHLCV 數據
- 市況分類器需要至少 50 天數據才能正確判斷趨勢
- 解決：用 Binance API 拿 90 天數據（但 HYPE 沒有）

### 2. 評分系統問題
- 31 天數據中，score 範圍只有 -20 到 +2
- 無法達到 ±25 的入場門檻
- 解決：降低門檻到 ±10

### 3. 信號分佈
- 門檻 ±5：80% 信號率（8 個 SELL，0 個 BUY）
- 門檻 ±10：70% 信號率（7 個 SELL，0 個 BUY）
- 門檻 ±15：60% 信號率（6 個 SELL，0 個 BUY）
- 門檻 ±20：10% 信號率（1 個 SELL，0 個 BUY）
- 門檻 ±25：0% 信號率（全部 NEUTRAL）

### 4. 做空信號為主
- 歷史數據是下跌趨勢，系統正確地給出做空信號
- 但沒有做多信號，代表在上漲趨勢中才會有 BUY

## 已完成的修改

### 1. 降低入場門檻
- 從 ±25 降到 ±10
- 盤整市況門檻從 8 降到 5

### 2. 增加做空信號
- RSI > 70（超買）時增加做空信號
- 資金費率 > 0.08% 時增加做空信號
- 動態調整做空門檻

### 3. 動態 TP/SL
- 高波動（ATR > 5%）：TP 2.5x / SL 1.5x
- 中波動（ATR 3-5%）：TP 2.0x / SL 1.5x
- 低波動（ATR < 3%）：TP 1.5x / SL 1.0x
- 盤整市況：TP/SL 收窄 20%

### 4. 市況分類器修復
- 數據門檻從 50 天降到 20 天
- SMA 週期根據數據長度自適應

## 2026-06-20 模擬倉系統建立

### 新增檔案
- `paper_trader.py` — 紙上交易模擬倉引擎
- `send_paper_report.py` — 模擬倉報告發送到 Telegram
- `hourly_check.py` — 每小時信號 + 模擬倉檢查
- `send_telegram.py` — Telegram bot 發送工具

### 模擬倉規則
- 保證金：$1000
- 槓桿：10x
- 入場條件：只進 STRONG_BUY / STRONG_SELL
- 每筆風險：5% 餘額（$50）
- 最大倉位：3 個同時持倉
- 倉位計算：size = $50 / SL距離；保證金上限 95% 帳戶
- 平倉條件：TP/SL 觸發、信號反轉、7 天超時

### 已修正問題
1. **TP/SL 方向 bug**：之前永遠用 'long' 計算，已修正為根據信號方向（BUY→long, SELL→short）傳入 side
2. **信號來源不一致**：paper_trader 原本用 signal_log 判斷信號、analyze_coin 計算 TP/SL，兩者可能不同步。已統一為直接跑 analyze_coin 取得信號和 TP/SL
3. **倉位大小顯示**：報告中加上倉位數量、名義價值、保證金

### Cron 架構
| 名稱 | 頻率 | 功能 | 發送目標 |
|------|------|------|----------|
| 每小時信號 + 模擬倉檢查 | 60 min | signal 變化通知 + 模擬倉建倉/平倉通知 | @TradingCommandersbot |
| 每日模擬倉報告 | 22:00 | 完整模擬倉绩效報告 | @TradingCommandersbot |
