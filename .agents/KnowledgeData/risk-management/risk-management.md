# 風險管理知識庫

> 來源：Van Tharp《Trade Your Way to Financial Freedom》、Ryan Jones《The Trading Game》、Nassim Taleb《Antifragile》、專業交易員經驗

---

## 一、核心風險管理原則

### 1. 永遠保護資本

**第一條規則：不要虧光**
- 單筆交易風險 ≤ 帳戶的 1-2%
- 總風險暴露 ≤ 帳戶的 6-10%
- 任何時候都要有「最壞情況」計劃

**數學基礎：**
```
虧損 10% → 需要賺 11% 回本
虧損 20% → 需要賺 25% 回本
虧損 50% → 需要賺 100% 回本
虧損 80% → 需要賺 400% 回本
```

**結論：**避免大虧比追求大賺更重要

---

### 2. 風險報酬比 (Risk-Reward Ratio, R:R)

```
R:R = 預期盈利 / 預期虧損

R:R = 1:1 → 需要 > 50% 勝率才能盈利
R:R = 1:2 → 需要 > 33% 勝率才能盈利
R:R = 1:3 → 需要 > 25% 勝率才能盈利
```

**期望值公式：**
```
E = (W × R) - (L × 1)

W = 勝率
R = 平均盈利（以風險單位計）
L = 敗率 = 1 - W

正期望值 = 長期盈利
```

**實例：**
- 勝率 40%，R:R = 1:3 → E = (0.4 × 3) - (0.6 × 1) = 0.6（正期望值）
- 勝率 60%，R:R = 1:1 → E = (0.6 × 1) - (0.4 × 1) = 0.2（正期望值）

## 二、倉位管理 (Position Sizing)

### 3. 固定百分比法 (Fixed Fractional)

```
倉位 = (帳戶 × 風險百分比) / 止損距離

範例：
帳戶 = $10,000
風險 = 1% = $100
止損距離 = $5
倉位 = $100 / $5 = 20 單位
```

**優點：**簡單、風險固定
**缺點：**不考慮市場波動率

---

### 4. ATR 波動率調整法

```
倉位 = (帳戶 × 風險百分比) / (ATR × 乘數)

範例：
帳戶 = $10,000
風險 = 1% = $100
ATR(14) = $2
止損 = 2 × ATR = $4
倉位 = $100 / $4 = 25 單位
```

**優點：**自動適應市場波動率
**驗證**：✅ Van Tharp 推薦

---

### 5. Kelly Criterion（凱利公式）

```
f* = (bp - q) / b

f* = 最佳倉位比例
b = 盈虧比（平均盈利/平均虧損）
p = 勝率
q = 1 - p = 敗率
```

**範例：**
- 勝率 55%，盈虧比 2:1
- f* = (2 × 0.55 - 0.45) / 2 = 0.325 = 32.5%

**實際使用：**
- 使用 25% Kelly（f*/4）= 8.1% 倉位
- 原因：Kelly 假設完美估計，實際需要保守

**驗證**：✅ 數學上最優，但需要精確估計勝率和盈虧比

---

### 6. 固定比率法 (Fixed Ratio)

```
每個合約所需保證金 = 基礎保證金 + (合約數 - 1) × delta

delta = 每增加一個合約需要的額外利潤
```

**範例：**
- 基礎保證金 = $10,000
- delta = $5,000
- 1 個合約：$10,000
- 2 個合約：$15,000
- 3 個合約：$20,000

**優點：**利潤增長時才增加倉位
**來源**：Ryan Jones《The Trading Game》

## 三、止損策略 (Stop Loss)

### 7. 固定百分比止損

```
止損 = 入場價 × (1 - 止損百分比)

多頭：止損 = 入場 × 0.95（5% 止損）
空頭：止損 = 入場 × 1.05
```

**優點：**簡單
**缺點：**不考慮市場結構

---

### 8. ATR 止損

```
多頭止損 = 入場價 - n × ATR
空頭止損 = 入場價 + n × ATR

常用：n = 2-3
```

**優點：**自適應波動率
**驗證**：✅ 最推薦的止損方法之一

---

### 9. 結構止損 (Structure Stop Loss)

```
多頭：止損設在近期波段低點下方
空頭：止損設在近期波段高點上方
```

**原則：**
- 止損設在「讓你的交易無效」的位置
- 如果價格回到止損，你的交易理由不再成立

---

### 10. 移動止損 (Trailing Stop)

**ATR 移動止損：**
```
多頭：止損 = 最高價 - n × ATR
空頭：止損 = 最低價 + n × ATR
只上移不下移（多頭）
```

**百分比移動止損：**
```
多頭：止損 = 最高價 × (1 - 追蹤百分比)
```

**Chandelier Exit（吊燈止損）：**
```
多頭：止損 = 最高價(22期) - 3 × ATR(22)
空頭：止損 = 最低價(22期) + 3 × ATR(22)
```

**來源**：Chuck LeBeau

---

### 11. 時間止損

```
如果 n 根 K 線後沒有達到目標 → 平倉

適用：
- 突破策略：突破後 3-5 根 K 線未延續
- 事件驅動：消息後未如預期反應
```

## 四、資金管理 (Money Management)

### 12. 風險層級

| 風險等級 | 單筆風險 | 總風險 | 適用 |
|---------|---------|--------|------|
| 保守 | 0.5% | 3% | 新手、大額帳戶 |
| 中等 | 1% | 6% | 有經驗交易者 |
| 積極 | 2% | 10% | 高信心、小帳戶 |
| 激進 | 3%+ | 15%+ | 不推薦 |

---

### 13. 最大回撤控制

```
回撤 < 10%：正常操作
回撤 10-15%：減少倉位 50%
回撤 15-20%：停止新交易，只平倉
回撤 > 20%：全面停止，重新評估策略
```

---

### 14. 相關性風險

```
避免同時持有高度相關的倉位

BTC 和 ETH 相關性 ≈ 0.8-0.9
→ 同時持有 BTC 和 ETH ≈ 只持有一個

分散原則：
- 不同相關性組別的倉位
- 相關性 < 0.5 才算真正分散
```

---

### 15. 槓桿管理

```
有效槓桿 = 總倉位價值 / 帳戶淨值

建議：
- 初學者：不使用槓桿
- 有經驗：最大 3x
- 專業：最大 5x
- 永遠不超過 10x

槓桿與止損的關係：
10x 槓桿 + 10% 止損 = 100% 虧損
5x 槓桿 + 10% 止損 = 50% 虧損
2x 槓桿 + 10% 止損 = 20% 虧損
```

## 五、組合風險管理

### 16. 投資組合熱度 (Portfolio Heat)

```
組合熱度 = Σ(每筆交易的風險百分比)

熱度 < 30%：安全
熱度 30-50%：注意
熱度 > 50%：過度風險
```

---

### 17. 風險價值 (Value at Risk, VaR)

```
VaR(95%) = 在 95% 信心水準下，一天內最大可能損失

計算方法：
1. 歷史法：根據歷史報酬分布
2. 參數法：假設常態分布
3. Monte Carlo 模擬
```

---

### 18. 壓力測試

```
測試策略在以下情況的表現：
1. 2020 年 3 月（COVID 崩盤）
2. 2022 年 5 月（LUNA/UST 崩潰）
3. 2022 年 11 月（FTX 崩潰）
4. 2018 年（加密寒冬）
5. 2017 年 12 月（BTC 泡沫）

如果策略在這些情況下回撤 > 50% → 需要調整
```

## 六、心理風險管理

### 19. 交易心理學原則

**常見心理陷阱：**

| 陷阱 | 描述 | 解決方案 |
|------|------|---------|
| 損失厭惡 | 虧損的痛苦是盈利快樂的 2 倍 | 接受虧損是交易的一部分 |
| 確認偏誤 | 只看支持自己觀點的資訊 | 主動尋找反對意見 |
| 沉沒成本 | 因為已經虧損而不願平倉 | 每筆交易獨立評估 |
| 過度自信 | 連續盈利後加大倉位 | 嚴格遵守倉位規則 |
| 報復交易 | 虧損後急於回本 | 虧損後強制休息 |
| FOMO | 錯過行情後追高 | 等待下一個機會 |
| 錨定效應 | 被入場價影響判斷 | 關注當前價格行為 |

### 20. 交易日誌

```
每筆交易記錄：
1. 日期時間
2. 標的
3. 方向（多/空）
4. 入場價
5. 止損價
6. 目標價
7. 倉位大小
8. 風險金額
9. 出場價
10. 盈虧
11. 交易理由
12. 情緒狀態（1-10）
13. 截圖
14. 覆盤筆記

每週回顧：
- 勝率
- 平均盈虧比
- 最大回撤
- 遵守規則的比例
- 心理狀態趨勢
```

## 七、特殊風險情境

### 21. 黑天鵝防護

```
1. 永遠不要全倉
2. 設置硬止損（不是心理止損）
3. 分散到不同交易所
4. 保留現金儲備
5. 定期提取利潤到冷錢包
6. 購買保險（如可用）
```

---

### 22. 交易所風險

```
風險類型：
1. 交易所倒閉（FTX 教訓）
2. 提款暫停
3. API 故障
4. 價格操縱
5. 監管風險

緩解：
- 不使用單一交易所
- 大額資金放冷錢包
- 定期提取利潤
- 選擇有儲備證明的交易所
```

---

### 23. 流動性風險

```
低流動性風險：
1. 滑價過大
2. 無法及時平倉
3. 價格操縱容易
4. 買賣價差大

判斷標準：
- 日成交量 > $10M：高流動性
- 日成交量 $1-10M：中等
- 日成交量 < $1M：低流動性（避免大倉位）
```

## 八、風險管理檢查清單

### 交易前檢查

- [ ] 這筆交易的風險 ≤ 帳戶的 2%？
- [ ] 有明確的止損位置？
- [ ] R:R ≥ 1:2？
- [ ] 總風險暴露 < 10%？
- [ ] 與現有倉位相關性不高？
- [ ] 不是情緒化交易？
- [ ] 有交易計劃？

### 每日檢查

- [ ] 所有倉位都有止損？
- [ ] 沒有超過最大倉位限制？
- [ ] 今日虧損沒有超過日限額？
- [ ] 情緒穩定？
- [ ] 交易日誌已更新？

### 每週檢查

- [ ] 本週回撤 < 5%？
- [ ] 遵守規則的比例 > 90%？
- [ ] 策略表現符合預期？
- [ ] 需要調整參數？
