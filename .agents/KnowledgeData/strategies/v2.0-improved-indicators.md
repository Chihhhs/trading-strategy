## 核心改良

### v1.0 的問題
- ❌ 過度依賴 EMA 單一指標（滯後、盤整期假信號）
- ❌ 沒有市場結構確認（HH/HL/LH/LL）
- ❌ 沒有成交量確認
- ❌ 沒有 BTC 相關性過濾
- ❌ ADX 實際用處有限

### v2.0 的改良
- ✅ EMA 只用於**趨勢方向判斷**，不用於精確入場
- ✅ 加入**市場結構**（HH/HL/LH/LL）作為首要確認
- ✅ 加入**成交量**確認突破/回調的有效性
- ✅ 加入**BTC 相關性**作為過濾器
- ✅ 砍掉 ADX，加入**資金費率**（加密特有）
- ✅ 因子權重重新分配

## 一、趨勢辨識（改良版）

### 多時間框架分析（MTF）
```
週線：判斷主要趨勢方向（只做順勢）
日線：確認市場結構（HH/HL 或 LH/LL）
4H：尋找入場區域（回調到 EMA20/50）
1H：精確入場時機（成交量確認）
```

### 趨勢判斷工具（改良版）

| 工具 | 用法 | 優先級 | 改良說明 |
|------|------|--------|---------|
| **市場結構** | HH+HL=上升趨勢，LH+LL=下降趨勢 | ⭐⭐⭐ 最高 | 比 EMA 更可靠，確認趨勢方向 |
| **EMA 20/50** | 價格 > EMA20 > EMA50 = 多頭排列 | ⭐⭐ 高 | 只用於確認方向，不用於找入場點 |
| **成交量** | 突破放量、回縮縮量 = 健康 | ⭐⭐ 高 | 確認價格變動的有效性 |
| **BTC 相關性** | BTC 穩住才做 Altcoin | ⭐⭐ 高 | 過濾掉逆勢操作 |
| **RSI(14)** | < 30 超賣，> 70 超買 | ⭐ 中 | 只在趨勢確認後用 |
| **資金費率** | > 0.1% 多頭擁擠，< -0.1% 空頭擁擠 | ⭐ 中 | 加密特有指標 |

### 砍掉的指標
| 指標 | 原因 |
|------|------|
| ADX | 計算複雜，實際用處有限，被市場結構取代 |
| MACD | 比 EMA 更滯後 |
| Ichimoku | 太多參數，加密市場效果不穩定 |
| SuperTrend | 盤整期假信號多 |

## 二、入場條件（改良版）

### 做多入場 — 需要 3/4 條件同時成立

```
必要條件（全部都要）：
✅ 1. 市場結構：HH（更高高點）+ HL（更高低點）
✅ 2. 趨勢確認：價格 > EMA20 > EMA50（多頭排列）
✅ 3. BTC 相關：BTC 穩定或上漲（不逆勢）

至少一個：
✅ 4a. 價格回調到 EMA20 附近（±2%）
✅ 4b. RSI < 40（回調到超賣區）
✅ 4c. 成交量縮量後放量（資金回流）

禁止入場：
❌ RSI > 70（超買）
❌ 資金費率 > 0.1%（多頭過於擁擠）
❌ BTC 正在下跌
```

### 做空入場 — 需要 3/4 條件同時成立

```
必要條件（全部都要）：
✅ 1. 市場結構：LH（更低高點）+ LL（更低低點）
✅ 2. 趨勢確認：價格 < EMA20 < EMA50（空頭排列）
✅ 3. BTC 相關：BTC 穩定或下跌

至少一個：
✅ 4a. 價格反彈到 EMA20 附近（±2%）
✅ 4b. RSI > 60（反彈到超買區）
✅ 4c. 成交量放大後縮量（資金流出）

禁止入場：
❌ RSI < 30（超賣）
❌ 資金費率 < -0.1%（空頭過於擁擠）
❌ BTC 正在上漲
```

### 分批入場策略（改良版）

```
第一批 30%：EMA20 附近入場（主要倉位）
第二批 40%：確認 HH/HL 後加倉
第三批 30%：突破關鍵位後加倉

條件：
- 只在盈利時加倉
- 每次加倉量遞減
- 加倉後移動止損
```

## 三、費波那契回撤（v2.1 新增）

### 核心概念

費波那契回撤用來找「回調到哪裡可以入場」。

```
上升趨勢中的回撤位：

100% ── 高點
  │
78.6% ── 強支撐（深回撤）
  │
61.8% ── ⭐ 黃金分割位
  │
50.0% ── 半回撤（心理支撐）
  │
38.2% ── ⭐ 最佳回撤入場點
  │
23.6% ── 最淺回撤
  │
  0% ── 低點
```

### 關鍵回撤位

| 回撤位 | 意義 | 入場優先級 |
|--------|------|----------|
| **38.2%** | 最常見的反彈點 | 🥇 最佳做多入場 |
| **50.0%** | 心理支撐 | 🥈 次佳做多入場 |
| **61.8%** | 黃金分割位 | 🥉 逆勢入場（風險較高） |
| **23.6%** | 最淺回撤 | ❌ 太貴，不做多 |

### 與 v2.0 策略的結合

```
四層過濾 + 費波那契：

1. BTC 方向 ✅
2. 市場結構 ✅
3. 價格位置：
   - 用費波那契位代替 EMA20
   - 上升趨勢 → 等回撤到 38.2% 或 50%
   - 下降趨勢 → 等反彈到 38.2% 或 50%
4. 確認信號 ✅
```

### 實例

**HYPE（30d 低點 $66.72，高點 $76.54）**
```
38.2%: $72.79  ← 你掛的 $72 就在這裡！
50.0%: $71.63  ← 接近 EMA20 $71.92
61.8%: $70.47  ← 你掛的 $70 在這裡！
```

**SOL（30d 低點 $72.76，高點 $75.60）**
```
38.2%: $74.51
50.0%: $74.18  ← 接近 EMA20 $73.96
61.8%: $73.44  ← 你入場 $73.14 在這裡！
```

**ETH（30d 低點 $1,736，高點 $1,843）**
```
38.2%: $1,802
50.0%: $1,790  ← 接近 EMA20 $1,788
61.8%: $1,777
```

### 入場規則

```
上升趨勢中：
- 價格回撤到 38.2% → 最佳做多入場
- 價格回撤到 50.0% → 次佳做多入場
- 價格在 23.6% 以上 → 太貴，不做多

下降趨勢中：
- 價格反彈到 38.2% → 最佳做空入場
- 價格反彈到 50.0% → 次佳做空入場

盤整中：
- 價格在 38.2% 或 50% 附近 → 小倉位嘗試
```

## 四、出場策略（改良版）

### 止損（SL）

```
方法 A：結構止損（推薦）
- 多頭：價格跌破前一個 HL（更高低點）
- 空頭：價格突破前一個 LH（更低高點）

方法 B：EMA 止損
- 多頭：價格收盤低於 EMA50
- 空頭：價格收盤高於 EMA50

方法 C：固定比例止損
- 多頭：入場 -5%（5x 槓桿）
- 空頭：入場 +5%（5x 槓桿）
```

### 獲利（TP）

```
TP1：+12%（7 次翻倍計劃目標）→ 平倉 50%
TP2：+20% → 平倉剩餘 50%

或：
TP1：前一個 HH（更高高點）
TP2：1.5-2 倍入場距離
```

## 四、量化因子模型（改良版）

### 因子權重重新分配

| 因子 | v1.0 權重 | v2.0 權重 | 改良原因 |
|------|----------|----------|---------|
| **市場結構** | — | **25%** | 新增，比 EMA 更可靠 |
| **動量** | 30% | **20%** | 降低，避免在趨勢市場過度依賴 |
| **成交量** | — | **15%** | 新增，確認價格變動有效性 |
| **BTC 相關性** | — | **15%** | 新增，過濾逆勢操作 |
| **情緒** | 15% | **10%** | 降低，FNG 是全市場指標 |
| **價值** | 20% | **10%** | 降低，加密市場數據不穩定 |
| **波動率** | 15% | **5%** | 降低，作為輔助即可 |
| **流動性** | 10% | — | 砍掉，對小幣種意義不大 |
| **鏈上** | 10% | — | 砍掉，數據取得成本高 |

### 新因子定義

#### 1. 市場結構（25%）
```
計算方式：
- 找出最近 3 個波段高點和 3 個波段低點
- HH（更高高點）+ HL（更高低點）= 多頭結構 → +25
- LH（更低高點）+ LL（更低低點）= 空頭結構 → -25
- 混合結構（HH+LL 或 LH+HL）= 盤整 → 0

應用：
- 多頭結構 → 只做多
- 空頭結構 → 只做空
- 盤整 → 不操作
```

#### 2. 動量（20%）
```
計算方式：
- 7d 報酬率（短期動量）
- 30d 報酬率（中期動量）
- 90d 報酬率（長期動量）

應用：
- 7d > +10% 且 30d > 0% → 短期強勢 → +20
- 7d < -10% 且 30d < 0% → 短期弱勢 → -20
- 7d > +20% 但 30d < 0% → 反彈但中期偏弱 → +5
- 7d < -20% 但 30d > 0% → 回調但中期偏強 → -5
```

#### 3. 成交量（15%）
```
計算方式：
- 7d 平均成交量 / 30d 平均成交量 = 量比

應用：
- 量比 > 1.5 且價格上漲 → 放量上漲 → +15
- 量比 > 1.5 且價格下跌 → 放量下跌 → -15
- 量比 < 0.7 且價格上漲 → 縮量上漲（無力）→ +5
- 量比 < 0.7 且價格下跌 → 縮量下跌（健康回調）→ -5
- 量比 0.7-1.5 → 正常 → 0
```

#### 4. BTC 相關性（15%）
```
計算方式：
- BTC 7d 報酬率方向

應用：
- BTC 7d > +3% → 做多 Altcoin → +15
- BTC 7d 在 -3% ~ +3% → 中性 → 0
- BTC 7d < -3% → 不做多 Altcoin → -15

注意：
- 這是過濾器，不是入場信號
- BTC 下跌時，即使 Altcoin 有做多信號也不入場
```

#### 5. 情緒（10%）
```
計算方式：
- FNG（恐懼貪婪指數）
- 資金費率

應用：
- FNG < 25 → 極度恐懼 → +10（可能是買入機會）
- FNG > 75 → 極度貪婪 → -10（可能是賣出機會）
- 資金費率 > 0.1% → 多頭擁擠 → -5
- 資金費率 < -0.1% → 空頭擁擠 → +5
```

#### 6. 價值（10%）
```
計算方式：
- 90d 報酬率（簡易版價值指標）

應用：
- 90d < -20% → 可能被低估 → +10
- 90d > +50% → 可能被高估 → -10
- 其他 → 0
```

#### 7. 波動率（5%）
```
計算方式：
- 7d ATR / 收盤價

應用：
- ATR% < 3% → 低波動 → +5（適合入場）
- ATR% > 8% → 高波動 → -5（風險大）
```

### 綜合評分公式

```
總分 = 市場結構×0.25 + 動量×0.20 + 成交量×0.15 + BTC相關×0.15 + 情緒×0.10 + 價值×0.10 + 波動率×0.05

評分範圍：-100 ~ +100

入場閾值：
- 做多：總分 > 40
- 做空：總分 < -40
- -40 ~ +40：不操作（盤整或信號不明）
```

## 五、實作範例

```python
def score_coin_v2(coin_data, btc_7d_change):
    score = 0
    
    # 1. 市場結構（25%）
    structure = coin_data.get('market_structure', 'neutral')
    if structure == 'bullish':    score += 25   # HH + HL
    elif structure == 'bearish':  score -= 25   # LH + LL
    else:                         score += 0    # 盤整
    
    # 2. 動量（20%）
    mom_7d = coin_data.get('momentum_7d', 0)
    mom_30d = coin_data.get('momentum_30d', 0)
    if mom_7d > 10 and mom_30d > 0:       score += 20
    elif mom_7d < -10 and mom_30d < 0:     score -= 20
    elif mom_7d > 20 and mom_30d < 0:      score += 5   # 反彈但中期偏弱
    elif mom_7d < -20 and mom_30d > 0:     score -= 5   # 回調但中期偏強
    
    # 3. 成交量（15%）
    vol_ratio = coin_data.get('vol_ratio_7d_30d', 1.0)
    price_7d = coin_data.get('price_change_7d', 0)
    if vol_ratio > 1.5 and price_7d > 0:    score += 15   # 放量上漲
    elif vol_ratio > 1.5 and price_7d < 0:  score -= 15   # 放量下跌
    elif vol_ratio < 0.7 and price_7d > 0:  score += 5    # 縮量上漲
    elif vol_ratio < 0.7 and price_7d < 0:  score -= 5    # 縮量下跌
    
    # 4. BTC 相關性（15%）
    if btc_7d_change > 3:       score += 15
    elif btc_7d_change < -3:   score -= 15
    
    # 5. 情緒（10%）
    fng = coin_data.get('fear_greed', 50)
    funding = coin_data.get('funding_rate', 0)
    if fng < 25:               score += 10
    elif fng > 75:             score -= 10
    if funding > 0.001:        score -= 5
    elif funding < -0.001:     score += 5
    
    # 6. 價值（10%）
    mom_90d = coin_data.get('momentum_90d', 0)
    if mom_90d < -20:          score += 10
    elif mom_90d > 50:         score -= 10
    
    # 7. 波動率（5%）
    atr_pct = coin_data.get('atr_pct', 0.05)
    if atr_pct < 0.03:         score += 5
    elif atr_pct > 0.08:       score -= 5
    
    return score

# 使用
coins = ['BTC', 'ETH', 'SOL', 'AAVE', 'HYPE']
btc_7d = 2.5  # BTC 7d +2.5%
for coin in coins:
    data = get_coin_data(coin)  # 從 API 獲取
    score = score_coin_v2(data, btc_7d)
    signal = '做多' if score > 40 else ('做空' if score < -40 else '觀望')
    print(f'{coin}: {score}/100 → {signal}')
```
