## 一、什麼是量化因子模型

**定義：** 用數學模型找出影響資產報酬的「因子」（Factors），根據這些因子的表現來做交易決策。

**核心邏輯：**
```
資產報酬 = α（超額報酬）+ β（市場報酬）+ ε（誤差）

因子模型：
報酬 = β₁×因子₁ + β₂×因子₂ + ... + βₙ×因子ₙ + α
```

**與技術分析的區別：**

| 維度 | 技術分析 | 量化因子模型 |
|------|---------|-------------|
| 基礎 | 價格形態/指標 | 統計/數學模型 |
| 主觀性 | 高（依賴判斷） | 低（規則明確） |
| 回測 | 困難（主觀規則） | 容易（客觀規則） |
| 適用範圍 | 單一資產 | 多資產/組合 |
| 數據需求 | K 線數據 | 大量歷史數據 |

## 二、常見因子類型

### 1. 動量因子（Momentum）

**邏輯：** 過去表現好的資產，未來繼續表現好（趨勢延續）

```
計算方式：
- 過去 12 個月報酬率（排除最近 1 個月）
- 過去 6 個月報酬率
- 過去 1 個月報酬率（短期動量）

應用：
- 買入過去 3-12 個月表現好的幣種
- 賣出過去 3-12 個月表現差的幣種
- 每月/每季再平衡

驗證：✅ 學術廣泛驗證（Jegadeesh & Titman 1993）
```

### 2. 價值因子（Value）

**邏輯：** 被低估的資產，長期會回歸合理價值

```
計算方式（加密市場）：
- MVRV Z-Score（市值/已實現市值）
- NVT Ratio（網絡價值/交易量）
- 市值/開發者活動比率
- 市值/社交媒體關注度

應用：
- MVRV < 0 → 買入（低估）
- MVRV > 7 → 賣出（高估）

驗證：✅ 學術驗證（Fama & French）| ⚠️ 加密市場仍在發展
```

### 3. 波動率因子（Volatility）

**邏輯：** 低波動資產風險調整後報酬更好（低波動異象）

```
計算方式：
- 過去 20 日/60 日/120 日波動率
- ATR / 收盤價
- 標準差

應用：
- 買入低波動幣種
- 賣出高波動幣種
- 或做空高波動幣種

驗證：✅ 學術驗證（低波動異象）
```

### 4. 流動性因子（Liquidity）

**邏輯：** 流動性差的資產需要更高的風險溢價

```
計算方式：
- 日成交量 / 市值（換手率）
- 買賣價差
- Amihud 非流動性指標

應用：
- 買入流動性改善的幣種
- 避開流動性持續下降的幣種

驗證：✅ 學術驗證
```

### 5. 規模因子（Size）

**邏輯：** 小市值資產長期報酬更高（但風險也更高）

```
計算方式：
- 市值排名
- 市值分位數

應用：
- 買入小市值幣種（高風險高報酬）
- 或做空大市值幣種（低風險低報酬）

驗證：✅ 學術驗證（Fama & French）| ⚠️ 加密市場小幣風險極高
```

### 6. 質量因子（Quality）

**邏輯：** 高質量（基本面好）的資產長期表現更好

```
計算方式（加密市場）：
- 開發者活動（GitHub commits）
- 活躍地址數
- 交易量/市值比
- 協議收入（DeFi）
- 團隊背景

應用：
- 買入基本面持續改善的幣種
- 避開基本面惡化的幣種

驗證：⚠️ 加密市場仍在發展
```

### 7. 情緒因子（Sentiment）

**邏輯：** 市場情緒極端時，反向操作

```
計算方式：
- 恐懼貪婪指數
- 社交媒體情緒（Twitter/Reddit）
- 資金費率
- 未平倉合約變化

應用：
- 極度恐懼 → 買入
- 極度貪婪 → 賣出

驗證：✅ 多次歷史驗證
```

### 8. 鏈上因子（On-Chain）

**邏輯：** 區塊鏈數據反映真實使用情況和持有者行為

```
計算方式：
- 活躍地址數變化
- 交易所淨流量
- 長期持有者持倉變化
- 礦工收入/算力
- 穩定幣供應量

應用：
- 交易所流出增加 → 看多
- 長期持有者增持 → 看多
- 穩定幣流入增加 → 看多

驗證：✅ Glassnode/CryptoQuant 數據驗證
```

## 三、因子組合策略

### 1. 多因子模型

```
綜合評分 = w₁×動量 + w₂×價值 + w₃×波動率 + w₄×流動性 + w₅×情緒 + w₆×鏈上

權重分配（範例）：
- 動量：30%
- 價值：20%
- 波動率：15%
- 流動性：10%
- 情緒：15%
- 鏈上：10%

應用：
- 每個幣種計算綜合評分
- 買入評分最高的 20% 幣種
- 賣出評分最低的 20% 幣種
- 每月再平衡
```

### 2. 因子輪動

```
根據市場環境調整因子權重：

牛市：
- 動量權重增加（趨勢強）
- 規模權重增加（小幣漲更多）
- 情緒權重減少（貪婪持續）

熊市：
- 價值權重增加（找低估）
- 波動率權重增加（風險管理）
- 情緒權重增加（恐懼是買入機會）

盤整：
- 流動性權重增加
- 質量權重增加
```

## 四、量化因子在加密市場的應用

### 1. 數據來源

| 數據類型 | 來源 | 費用 |
|---------|------|------|
| K 線數據 | Binance API / CCXT | 免費 |
| 鏈上數據 | Glassnode / CryptoQuant | 付費 |
| 社交數據 | Santiment / LunarCrush | 付費 |
| 基本面 | CoinGecko / CoinMarketCap | 免費 |

### 2. 實作範例

```python
# 簡化版多因子評分
def score_coin(coin_data):
    score = 0
    
    # 動量因子（30%）
    momentum = (coin_data['price_30d'] / coin_data['price_90d'] - 1) * 100
    score += momentum * 0.3
    
    # 價值因子（20%）
    mrvv = coin_data['mrvv_zscore']
    if mrvv < 0: score += 20 * 0.2
    elif mrvv > 7: score -= 20 * 0.2
    
    # 波動率因子（15%）
    vol = coin_data['volatility_30d']
    if vol < 0.5: score += 15 * 0.15
    elif vol > 2: score -= 15 * 0.15
    
    # 情緒因子（15%）
    fng = coin_data['fear_greed_index']
    if fng < 20: score += 15 * 0.15
    elif fng > 80: score -= 15 * 0.15
    
    # 鏈上因子（20%）
    exchange_flow = coin_data['exchange_netflow']
    if exchange_flow < 0: score += 20 * 0.2  # 流出 = 看多
    
    return score
```

### 3. 回測框架

```
步驟：
1. 收集歷史數據（至少 1 年）
2. 計算每個因子的歷史值
3. 標準化因子值（Z-Score）
4. 計算綜合評分
5. 根據評分排序，選擇前 20% 買入
6. 定期再平衡（每月/每季）
7. 計算報酬率、最大回撤、夏普比率
```

## 五、量化因子模型的優缺點

### 優點

```
1. 客觀：規則明確，不受情緒影響
2. 可回測：可以用歷史數據驗證
3. 可擴展：可以同時分析多個幣種
4. 紀律性：嚴格執行策略
5. 可優化：可以調整因子權重
```

### 缺點

```
1. 數據需求：需要大量歷史數據
2. 過擬合風險：歷史表現不代表未來
3. 因子失效：某些因子可能長期失效
4. 執行成本：手續費、滑價
5. 黑天鵝：極端事件可能導致重大損失
```

## 六、對你做決策的幫助

### 直接幫助

```
1. 系統化決策：不用靠感覺，有明確的買入/賣出信號
2. 風險管理：通過因子分散風險
3. 回測驗證：可以用歷史數據驗證策略
4. 多維度分析：不只看技術面，還看基本面、情緒、鏈上數據
```

### 間接幫助

```
1. 理解市場：了解什麼因素驅動價格
2. 避免偏見：減少確認偏誤、錨定效應
3. 長期視角：因子投資通常是長期的
4. 組合管理：可以管理多個幣種的組合
```

### 建議

```
1. 從簡單開始：先做動量 + 情緒因子
2. 逐步增加：加入價值、鏈上因子
3. 定期回測：每月檢視因子表現
4. 動態調整：根據市場環境調整因子權重
5. 風險第一：永遠把風險管理放在第一位
```
