# 市場微觀結構知識

> 來源：Maureen O'Hara《Market Microstructure Theory》、Larry Harris《Trading and Exchanges》

---

## 一、訂單簿動態

### 1. 訂單簿基礎

```
Bid（買方）：買家願意買的最高價格
Ask（賣方）：賣家願意賣的最低價格
Spread（價差）= Ask - Bid
Mid Price = (Bid + Ask) / 2
```

**價差的意義：**
- 價差小 → 流動性好，交易成本低
- 價差大 → 流動性差，交易成本高
- 加密貨幣主要交易對價差通常 < 0.1%

---

### 2. 訂單簿不平衡 (Order Book Imbalance)

```
OBI = (Bid Volume - Ask Volume) / (Bid Volume + Ask Volume)

OBI > 0.3：買方主導 → 價格可能上漲
OBI < -0.3：賣方主導 → 價格可能下跌
```

**用途：**短期價格方向預測（秒級到分鐘級）

---

### 3. 大單分析 (Large Order Detection)

```
冰山訂單：大單拆分成小單逐步執行
判斷方法：
- 同一價格持續有成交但掛單不減少
- 成交量大但訂單簿深度不變
```

**追蹤大戶動向：**
- 交易所大額轉帳 → 可能準備交易
- Whale Alert 監控
- 鏈上數據分析

---

### 4. 流動性與滑價

```
預期滑價 = 0.5 × Spread + 衝擊成本

衝擊成本 ∝ (訂單量 / 市場深度)²

深度圖分析：
- 深度均勻 → 大單容易執行
- 深度不均 → 可能在某價格產生大幅滑價
```

## 二、市場結構

### 5. 市場週期

```
四個階段：
1. 累積（Accumulation）：底部，聰明錢買入
2. 上漲（Mark Up）：趨勢上升，散戶開始注意
3. 分銷（Distribution）：頂部，聰明錢賣出
4. 下跌（Mark Down）：趨勢下降，散戶恐慌

對應的成交量特徵：
- 累積：低量，價格波動小
- 上漲：量增價漲
- 分銷：高量但價格停滯
- 下跌：恐慌性放量
```

---

### 6. 支撐與阻力

**形成原因：**
1. 歷史價格記憶（之前的高/低點）
2. 心理整數位（$10,000、$100,000）
3. 成交量密集區（Volume Profile POC）
4. 移動平均線
5. Fibonacci 回調位

**強度判斷：**
- 測試次數越多 → 越強
- 成交量越大 → 越強
- 時間越久 → 越強
- 突破時的成交量 → 確認突破有效性

**角色轉換：**
- 突破的阻力 → 變成支撐
- 跌破的支撐 → 變成阻力

---

### 7. 成交量分析

**基本原則：**
- 價格上漲 + 成交量增加 → 趨勢確認
- 價格上漲 + 成交量減少 → 趨勢衰弱
- 價格下跌 + 成交量增加 → 恐慌性賣出
- 價格下跌 + 成交量減少 → 賣壓減少，可能反轉

**異常成交量：**
- 成交量 > 平均 3 倍 → 重大事件/轉折點
- 成交量極低 → 盤整/方向選擇前

---

| 形態 | 含義 | 可靠性 |

## 三、市場指標

### 9. 市場廣度指標

```
上漲/下跌比率 = 上漲數量 / 下跌數量
> 1.5：市場過熱
< 0.5：市場超賣

新高新低比率 = 52週新高 / 52週新低
> 1：多頭市場
< 1：空頭市場
```

---

### 10. 波動率指數

**加密貨幣：**
- BVOL（BitMEX 波動率指數）
- DVOL（Deribit 波動率指數）

**解讀：**
- VIX/BVOL 上升 → 恐懼增加 → 可能見底
- VIX/BVOL 下降 → 自滿增加 → 可能見頂
- 極端高值 → 恐慌性底部
- 極端低值 → 自滿性頂部

---

### 11. 資金費率（加密貨幣特有）

```
資金費率 > 0：多頭支付空頭 → 市場看多
資金費率 < 0：空頭支付多頭 → 市場看空

極端值：
> 0.1%（8h）：極度看多 → 可能回調
< -0.1%（8h）：極度看空 → 可能反彈
```

---

## 四、執行策略

### 13. 最佳執行

**限價單 vs 市價單：**
```
限價單：
+ 控制價格
+ 較低手續費（maker fee）
- 可能不成交

市價單：
+ 保證成交
- 滑價風險
- 較高手續費（taker fee）
```

**大單執行策略：**
1. TWAP（時間加權平均價格）：分時段執行
2. VWAP（成交量加權平均價格）：跟隨成交量分佈
3. Iceberg：冰山訂單
4. POV（參與率）：固定市場參與比例

---

### 14. 滑價管理

```
預期滑價 = f(訂單量, 流動性, 波動率)

降低滑價：
1. 使用限價單
2. 在流動性高的時段交易
3. 拆分大單
4. 避免新聞事件期間交易
5. 選擇深度好的交易對
```
