## 二、風險事件預警信號

### 3. 交易所風險信號

```
⚠️ 以下信號可能預示交易所問題：
1. 提款延遲或暫停
2. 異常的大額轉帳
3. 交易所儲備證明異常
4. 社交媒體大量投訴
5. 員工離職潮
6. 監管調查消息
7. 透明度下降
8. 交易異常（閃崩、插針）
```

**應對：**
```
發現信號後：
1. 立即提取大部分資金
2. 保留小額用於交易
3. 監控情況發展
4. 如果惡化，全部撤出
```

---

### 4. 穩定幣風險信號

```
⚠️ 以下信號可能預示穩定幣問題：
1. 脫鉤 > 1%
2. 脫鉤持續時間 > 1 小時
3. 贖回困難
4. 儲備證明異常
5. 審計報告延遲
6. 監管調查
7. 大額拋售

歷史案例：
- UST 2022-05：脫鉤 → 死亡螺旋 → $40B 蒸發
- USDC 2023-03：短暫脫鉤到 $0.88（矽谷銀行事件）
- DAI 2023-03：短暫脫鉤
```

**應對：**
```
穩定幣脫鉤 > 2%：
1. 立即兌換為其他穩定幣或法幣
2. 不要試圖「抄底」脫鉤的穩定幣
3. 等待穩定後再重新配置
```

---

### 5. 智能合約風險信號

```
⚠️ 以下信號可能預示合約問題：
1. 未經審計的新協議
2. 管理員權限過大
3. 可升級合約（upgradeable）
4. 複雜的數學模型
5. 高 APY 承諾（>100%）
6. 團隊匿名
7. 代碼未開源
8. 無漏洞賞金計劃
```

## 三、風險事件應對框架

### 6. 緊急應對流程

```
發現風險信號：

第一步：評估（5 分鐘內）
□ 信號是什麼？
□ 影響範圍？
□ 緊急程度？

第二步：保護資本（15 分鐘內）
□ 減少受影響的倉位
□ 轉移資金到安全地方
□ 設定更緊的止損

第三步：分析（1 小時內）
□ 查閱多個信息來源
□ 評估是否是誤報
□ 確定影響持續時間

第四步：決策
□ 觀望（如果風險可控）
□ 部分減倉（如果不確定）
□ 全部離場（如果風險嚴重）

第五步：覆盤
□ 記錄事件經過
□ 總結教訓
□ 更新風控規則
```

---

### 7. 壓力測試

```
定期測試以下情境：

情境 1：BTC 下跌 30%
- 你的倉位會怎樣？
- 是否會觸發清算？
- 有多少保證金緩衝？

情境 2：交易所暫停提款
- 你的資金是否安全？
- 有多少資金在交易所？
- 是否有備用方案？

情境 3：穩定幣脫鉤
- 你持有多少穩定幣？
- 是否會受到影響？
- 是否有替代方案？

情境 4：重大協議漏洞
- 你的資金是否在受影響協議？
- 是否可以及時撤出？
- 是否有保險？
```

## 四、風險分散策略

### 8. 交易所分散

```
原則：不要把所有資金放在一個交易所

建議配置：
- 主要交易所 1：50%（如 Hyperliquid）
- 主要交易所 2：30%（如 Binance）
- 冷錢包：20%（長期持有）

分散原因：
1. 交易所可能倒閉
2. 交易所可能暫停提款
3. 交易所可能被駭
4. 監管風險
```

---

### 9. 資產分散

```
原則：不要過度集中在單一資產

建議配置（激進型）：
- BTC：40%
- ETH：20%
- 穩定幣：20%
- Altcoins：15%
- 新倉位機會：5%

建議配置（保守型）：
- BTC：50%
- ETH：25%
- 穩定幣：25%
```

---

### 10. 時間分散（DCA）

```
原則：不要一次性投入所有資金

DCA（Dollar Cost Averaging）：
- 將資金分成 4-8 份
- 每週或每兩週投入一份
- 降低入場時機風險

適用場景：
- 長期看好某個幣種
- 不確定短期走勢
- 想降低平均成本
```

## 五、保險與對沖

### 11. 對沖策略

```
對沖目的：降低整體風險，不是為了盈利

方法 1：多空對沖
- 做多 BTC + 做空 ETH
- 如果 BTC 漲，做多賺錢
- 如果 ETH 跌，做空賺錢
- 淨效果：降低市場波動影響

方法 2：期權對沖
- 買入 Put 期權（下跌保險）
- 成本：權利金
- 效果：限制最大虧損

方法 3：穩定幣對沖
- 部分倉位轉為穩定幣
- 降低整體波動
- 保留資金等待機會
```

---

### 12. 保險協議

```
加密貨幣保險協議：
- Nexus Mutual：智能合約保險
- InsurAce：DeFi 保險
- Unslashed：交易所風險保險

保險類型：
1. 智能合約保險：保障合約漏洞
2. 交易所保險：保障交易所風險
3. 穩定幣保險：保障脫鉤風險

注意：
- 保險有成本（保費）
- 理賠可能需要時間
- 保險公司本身也有風險
```

### 14. 代幣解鎖監控

```
大額解鎖可能導致價格下跌

監控工具：
- Token Unlocks：tokenunlocks.app
- CryptoRank：cryptorank.io

交易策略：
- 解鎖前 1-2 周：減少倉位
- 解鎖當天：觀望
- 解鎖後：如果拋壓不大，可以考慮買入
```

## 七、個人風險管理清單

### 每日檢查

```
□ 所有倉位都有止損？
□ 保證金使用率 < 80%？
□ 沒有過度集中在單一資產？
□ 交易所沒有異常？
□ 穩定幣沒有脫鉤？
□ 沒有重大新聞事件？
```

### 每週檢查

```
□ 本週回撤 < 5%？
□ 遵守規則的比例 > 90%？
□ 策略表現符合預期？
□ 需要調整參數？
□ 有新的風險信號？
```

### 每月檢查

```
□ 本月總盈虧？
□ 最大回撤？
□ 夏普比率？
□ 策略是否需要調整？
□ 風險管理規則是否需要更新？
```
